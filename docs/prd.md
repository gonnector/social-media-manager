# Social Media Manager — PRD (Product Requirements Document)

> 프로젝트 오너: Dylan (Gonnector)
> PM: JARVIS
> 초안 작성: 2026-03-22
> 상태: **초안 — Dylan 리뷰 대기**

---

## 1. 개요

멀티 소셜미디어 플랫폼에 한국어(원본) + 영어(번역) 콘텐츠를 동시 자동 포스팅하고, 퍼포먼스를 분석·최적화하는 도구.

---

## 2. 대상 플랫폼

| 플랫폼 | API 상태 | 비용 | 우선순위 |
|--------|---------|------|----------|
| Threads (Meta) | 무료, 250건/일 | $0 | Phase 1 |
| X (Twitter) | Free~Pay-Per-Use | $0~$4.50/월 | Phase 1 |
| LinkedIn | Dev 티어 500콜/일 | $0 | Phase 1 |
| Facebook | Marketing API | 확인 필요 | Phase 2 |
| Instagram | Graph API | 확인 필요 | Phase 2 |

---

## 3. 기능 요건

### 3-1. 포스팅

| 요건 | 상세 | 우선순위 |
|------|------|----------|
| 크로스 플랫폼 동시 포스팅 | 하나의 콘텐츠를 여러 플랫폼에 동시 게시 | P0 |
| 한국어 + 영어 동시 | 한글 원본 + AI 영문 번역 각각 포스팅 | P0 |
| 플랫폼별 자동 커스터마이징 | 글자 수 제한, 포맷, 해시태그 규칙 자동 적용 | P0 |
| 텍스트 포스팅 | 본문 텍스트 | P0 |
| 이미지 첨부 | 단일/멀티 이미지 | P0 |
| 동영상 첨부 | 동영상 업로드 | P1 |
| 하이퍼링크 | URL 포함 및 OG 미리보기 | P1 |
| 긴 글 | Threads 텍스트 첨부(10K자), LinkedIn Article, X 장문 | P1 |
| 해시태그 | 플랫폼별 최적 해시태그 자동 생성 | P2 |
| 고유 태그/카테고리 | Threads 토픽 등 플랫폼 고유 분류 | P2 |

### 3-2. 예약 및 반복

| 요건 | 상세 | 우선순위 |
|------|------|----------|
| 예약 포스팅 | 특정 시간에 자동 게시 | P1 |
| 반복 포스팅 | 동일 콘텐츠 주기적 재게시 (설정 가능) | P2 |
| 최적 시간 추천 | 퍼포먼스 데이터 기반 게시 시간 제안 | P2 |

### 3-3. 분석

| 요건 | 상세 | 우선순위 |
|------|------|----------|
| 플랫폼별 메트릭 수집 | 조회수, 좋아요, 댓글, 공유, 클릭 | P1 |
| 크로스 플랫폼 비교 | 동일 콘텐츠의 플랫폼별 퍼포먼스 비교 | P1 |
| 인게이지먼트 트렌드 | 시간대별, 요일별, 콘텐츠 유형별 분석 | P2 |
| AI 인사이트 | 퍼포먼스 기반 자동 최적화 제안 | P2 |

### 3-4. 관리

| 요건 | 상세 | 우선순위 |
|------|------|----------|
| 포스트 편집 | 게시 후 수정 (플랫폼별 제약 반영) | P1 |
| 일괄 삭제 | 여러 포스트 한 번에 삭제 | P1 |
| 콘텐츠 히스토리 | 게시 이력, 수정 이력 추적 | P2 |

---

## 4. 콘텐츠 입력

### 단기 (Phase 1)
- Dylan이 직접 전달 (텍스트, 이미지, 동영상)
- 주로 영어 포스팅 위주

### 장기 (Phase 2+)
- CDP (Contents Data Platform)에서 자동 가져오기
- CDP → 번역 → 포스팅 전체 자동화 파이프라인

---

## 5. 아키텍처 (권장안)

```
콘텐츠 입력 (Dylan 직접 / CDP)
    ↓
AI 번역 (Claude — 플랫폼별 톤·길이·제약 최적화)
    ↓
플랫폼 어댑터 (각 API의 차이를 추상화)
  ├── ThreadsAdapter
  ├── XAdapter
  ├── LinkedInAdapter
  ├── FacebookAdapter (Phase 2)
  └── InstagramAdapter (Phase 2)
    ↓
포스팅 큐 (예약, 재시도, rate limit 관리)
    ↓
공식 API 포스팅
    ↓
퍼포먼스 수집 (각 플랫폼 Insights/Analytics API)
    ↓
분석 대시보드 + AI 최적화 제안
```

### 핵심 설계 원칙
- **어댑터 패턴** — 플랫폼 추가 시 새 어댑터만 구현
- **API 버전 핀닝** — 플랫폼별 breaking change 격리
- **토큰 자동 갱신** — 만료 7일 전 크론잡
- **큐잉 + 재시도** — rate limit 429 응답 시 지수 백오프

| 서버/인프라 | ~$5~$20 |
| **운영 비용 소계** | **~$15~$40/월** |

## 7. 플랫폼별 제약 요약

| 항목 | Threads | X | LinkedIn |
|------|---------|---|----------|
| 글자 제한 | 500자 (첨부: 10K) | 25K (Premium) | 사실상 무제한 |
| 이미지 | 20장, URL 호스팅 | 4장, 청크 업로드 | 20장, 2단계 업로드 |
| 동영상 | 1GB, 5분 | 512MB | 확인 필요 |
| 수정 | ❌ (삭제→재발행) | 30분/5회 | 본문만 |
| 분석 | views, likes, replies | public+non-public(30일) | impression, reach, reaction |

## 8. 리스크

| 리스크 | 영향 | 대응 |
|--------|------|------|
| Threads 앱 리뷰 거절 | Phase 1 지연 | 스크린캐스트 품질 강화 + 브라우저 자동화 폴백 |
| X API 가격 변경 | 비용 급증 | Pay-Per-Use 예산 상한 + SaaS 전환 옵션 |
| 3개 API 동시 breaking change | 포스팅 중단 | 어댑터 패턴 격리 + Changelog RSS 모니터링 |
| OAuth 토큰 갱신 실패 | 자동화 중단 | 자동 갱신 크론잡 + 실패 알림 |
| 콘텐츠 스팸 정책 위반 | 계정 제한 | 플랫폼별 정책 숙지, 크로스 포스팅 차별화 |

## 9. 마일스톤

### Phase 1 — MVP (Threads + X + LinkedIn)
- Threads/X/LinkedIn 개발자 계정 셋업
- 공식 API 기반 텍스트+이미지 포스팅
- AI 번역 파이프라인 (Claude)
- 플랫폼별 자동 커스터마이징
- 기본 퍼포먼스 메트릭 수집

### Phase 2 — 확장
- Facebook + Instagram 추가
- 예약 포스팅
- 분석 대시보드
- 일괄 삭제/편집
- CDP 연동

### Phase 3 — 고도화
- AI 인사이트 (최적 시간, 해시태그, 콘텐츠 변형)
- 반복 포스팅
- A/B 테스팅 지원
